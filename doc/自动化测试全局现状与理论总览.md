# 自动化测试全局现状与理论总览

更新时间：2026-04-22

## 1. 文档目标

这份文档回答四个问题：

1. `auto_test` 现在到底覆盖了哪些测试能力。
2. 当前进度到了哪个阶段，哪些已经落地、哪些仍在验证。
3. 自动化评估背后的理论框架是什么（不仅是压缩专项）。
4. 现阶段结果能否支持“统计结论”与“策略决策”。

## 2. 项目全局定位

`auto_test` 是 AI 会话能力的自动化验证底盘，不是单一脚本。  
它的目标是把“多轮对话测试 -> 结构化评估 -> 报告解释”做成可重复、可审计、可扩展体系。

当前覆盖四类问题：

1. 执行可靠性：会话是否稳定跑完，链路是否健康。
2. 通用对话质量：任务完成、指令遵循、连贯性、安全性、工具正确性。
3. 探针能力：针对关键事实/状态进行定点追问，判断是否保持。
4. 压缩专项：识别压缩完成并评估压缩前后记忆保持变化。

## 3. 全局能力版图（当前）

### 3.1 执行与采集层

能力：自动 create/execute 会话、SSE 事件解析、原始事件落盘、workspace 导出。  
状态：已可稳定批量运行；偶发上游网络或服务波动会造成中断。

### 3.2 用户模拟层

能力：角色化用户模拟、能力模式切换、首轮兜底策略。  
状态：已作为默认输入源使用，首轮由模拟器优先生成，兜底文案仅在失败时触发。

### 3.3 通用评估层（LLM-only）

能力：  

1. 主评估输出（`evaluation_primary`）
2. 影子评估结构（`evaluation_v2_shadow`）
3. 模式对比（`evaluation_compare`，用于 A/B 观察）

状态：已落地，可在不破坏历史字段的前提下并行演进。

### 3.4 Probe 评估层

能力：  

1. 确定性探针（deterministic）
2. 主观探针（llm judge）
3. 混合加权聚合

状态：已落地，支持数据集驱动与结构化输出。

### 3.5 压缩专项层（作为 profile/专题能力）

能力：  

1. 压缩完成识别
2. 压缩前后窗口切分
3. 分层保持（must/should/may）分析
4. 中文可解释报告

状态：流程与报告已跑通，统计稳定性仍在提升阶段。

## 4. 当前阶段进度（里程碑视角）

### 已完成（可用）

1. 多轮自动会话执行与结构化结果落盘。
2. LLM-only 评估主链路。
3. Probe 双模式（deterministic + llm）评估。
4. 评估模式对比输出（A/B 观察）。
5. 多 profile 路由与组合评估（影子结构）。
6. 压缩扫描与中文后处理报告能力。

### 已验证但仍在优化

1. 压缩完成覆盖率（并非每个 session 都能进入“已验证压缩完成”）。
2. 批量运行稳定性（run_error 受上游波动影响）。
3. 从“流程验证级”升级到“决策支撑级”所需的有效样本积累。

## 5. 理论框架（全局，不仅压缩）

## 5.1 三层验证理论

我们把测试判断拆成三层，避免混淆：

1. 健康层：系统有没有稳定完成测试流程。
2. 完成层：关键机制是否真实发生（例如压缩是否完成）。
3. 效果层：在机制已完成前提下，观察业务效果（如记忆保持）。

结论规则：上层依赖下层。  
如果完成层证据不足，效果层结论自动降级为“仅参考”。

## 5.2 通用质量理论

通用评估遵循“底盘 + profile”的思想：

1. 底盘维度：任务完成、指令遵循、连贯性、安全性、工具正确性。
2. profile 维度：面向场景的专项能力（压缩只是其中之一）。
3. 输出策略：保留统一总分，同时保留结构化分项，便于审计与回归。

## 5.3 Probe 理论

Probe 的作用是“定点验收”，不是泛化印象评分。  
通过规则判定与主观判定并行，可以同时兼顾稳定性与语义理解能力。

## 5.4 样本与统计意义理论

“跑了很多轮”不等于“统计可信”。  
统计结论需要同时满足：

1. 足够的有效 session（跨会话可重复）。
2. 足够的后机制样本（例如后压缩探针样本）。
3. 足够稳定的链路健康（低超时/低中断/低异常）。

## 7. 结论等级定义（对外沟通建议）

### 7.1 流程验证级

含义：证明链路和机制可运行。  
适用：内部验收、工程联调。  
限制：不用于最终业务决策。

### 7.2 趋势观察级

含义：可观察方向性变化。  
适用：迭代优先级排序。  
限制：仍需扩大样本确认。

### 7.3 决策支撑级

含义：可用于较稳定的方案优选。  
适用：版本定版与策略落地。  
前提：高有效 session + 稳定健康指标 + 复现实验一致。

当前整体处于“流程验证级 -> 趋势观察级”的过渡阶段。

## 8. 当前主要风险（全局）

1. 上游服务波动导致 run_error，降低有效样本比例。
2. 有效 session 与总 session 存在差距，易被误解为“样本充足”。
3. 统计口径容易混用（独立口径 vs 展开口径），造成结论偏差。
4. 专项结果（如压缩）容易被误读为全局质量结论，需要分层解释。

## 9. 下一阶段目标（全局）

1. 提升批量执行稳定性，降低 run_error 比率。
2. 提升关键机制完成覆盖率（如压缩完成验证覆盖率）。
3. 按统一口径扩展有效样本，推动结论等级升级。
4. 完善“全局质量 + 专项 profile”联合解读模板，统一对外口径。
