### 一、 核心对话与记忆能力验证

- 

##   **实验目标 1：验证长期依赖与状态跨轮延续能力**

```
**实验目的：** 确保模型在经过大量对话轮次后，依然能够记住早期的关键设定和约束条件 。
```

  - 
**原因说明：** 实际应用中需要避免模型忘记之前的约束、重复执行做过的步骤或者出现上下文断片 。通过在第1轮设定问题或约束，并在第50轮之后再次提及，可以直观检验长期记忆的有效性 。

- 

##   **实验目标 2：验证跨轮次分散事实融合推理能力**

```
**实验目的：** 评估模型能否从多轮历史对话中提取并整合碎片化的信息，从而得出正确结论 。
```

  - 
**原因说明：** 如果模型压缩记忆不当，容易导致只抓到局部信息、推理链断裂或结论片面 。可以设计此类实验：在第50轮提问，但正确回答该问题必须同时用到第1轮、第20轮和第30轮的信息 。

- 

##   **实验目标 3：验证多主题上下文路由（Topic Shift）处理能力**

```
**实验目的：** 测试对话主题发生较大跳跃时记忆系统的鲁棒性，以及跨主题的回溯连贯性 。
```

  - 
**原因说明：** 当主题变化较大时，前序主题的 summary 可能会对当前对话产生负面影响，发生“主题污染”或把 B 主题信息误用到 A 主题 。此外，用户在多轮其他主题后回到最初主题时，系统需要能够顺畅接上上下文 。

- 

##   **实验目标 4：验证结论的可回溯证据能力**

```
**实验目的：** 确保模型基于压缩后的 summary 做出的回答和推理，是具备客观依据且可以溯源的 。
```

  - 
    **原因说明：** 需要防范和评估模型出现“给结论不给依据”、“依据对不上”或者“无法追溯原文”等失败表现 。

### 二、 系统边界与性能稳定性验证

- 

##   **实验目标 5：测试记忆的轮次极限与遗忘分布**

```
**实验目的：** 探底当前记忆压缩机制的容量物理极限，描绘出信息开始遗忘的轮次分布图 。
```

  - 
**原因说明：** 了解在达到多少轮次时模型会开始丢失关键信息，有助于为后续的系统设计（如 summary 满载策略）提供关键的数据参考 。

- 

##   **实验目标 6：验证压缩比的遵循能力与分布差异**

```
**实验目的：** 检验大模型对设定的“目标压缩比”（如 0.8）的执行精确度，以及不同阈值设置下的实际压缩效果 。
```

  - 
**原因说明：** 每次压缩的目的是为了控制结果大小，如果压缩后的长度与触发阈值接近，则压缩失效 。需要通过多次测试观察压出来的长度分布是否有显著差异，以及均值是否与预设的压缩率保持一致 。

- 

##   **实验目标 7：验证预算可控与性能稳定性**

```
**实验目的：** 监控引入压缩机制后，系统的响应表现和 token 消耗是否稳定 。
```

  - 
    **原因说明：** 为了避免在实际运行中出现时延抖动、token 异常飙升，或者在触发压缩的临界轮次出现系统崩溃的情况 。

### 三、 自动化实验与对照评估体系建立

- 

##   **实验目标 8：建立标准化的批量对照实验与评估体系**

```
**实验目的：** 通过自动化脚本进行批量跑测，使用具体指标（如通过率）来科学评估压缩机制的效果 。
```

  -   
    **原因说明：** 单一 case 很难充分说明问题，需要批量测试来寻找未通过 case 的共性特征 。实验规范要求必须建立基线对照：将所需信息直接放入最近轮次（模拟 Full context），与使用大模型压缩后 summary 的效果进行对比，并可引入大模型自身来自动化评估对话流程及统计准确率 。
